# 投资决策流程

## 概述
投资决策流程是 VC/PE 基金从项目评估到最终拍板的核心环节，由投资决策委员会（Investment Committee, IC）主导。规范的投决流程能确保投资决策的质量和一致性，降低个人偏见和情绪化决策的风险。IC Memo（投资备忘录）是投决会的核心文件，其质量直接影响决策效果。

## 核心内容

### 投决会（Investment Committee）机制

#### IC 的组成

| 角色 | 职责 | 典型人数 |
|------|------|----------|
| Managing Partner / CIO | 最终决策权，对基金业绩负总责 | 1-2 人 |
| Partners / Principals | 各赛道/阶段的负责人，汇报项目情况 | 3-5 人 |
| CFO / 合规负责人 | 从财务和合规角度提供意见 | 0-1 人 |
| 外部顾问（可选） | 特定行业/领域的专业意见 | 0-2 人 |

**IC 的规模：**
- 早期 VC：通常 3-5 人
- 成长期 VC/PE：通常 5-7 人
- 大型 PE：通常 7-10 人，可能设多个分委员会

#### IC 决策规则

| 决策规则 | 说明 | 适用场景 |
|----------|------|----------|
| 全票通过（Unanimous） | 所有 IC 成员一致同意 | 最严格，适用于大型 PE |
| 多数通过（Majority） | 超过半数同意 | 最常见 |
| 关键人士一票否决 | 特定 Key Person 有权否决 | 部分基金的治理安排 |
| 金额分级决策 | 不同投资金额对应不同决策层级 | 大型基金的分级管理 |

#### 投决会流程

```
Step 1: 项目团队提交 IC Memo
         ↓
Step 2: IC 成员提前审阅（通常提前 2-3 天）
         ↓
Step 3: 投决会会议
         ├── 项目团队 Presentation（20-30 分钟）
         ├── Q&A 和讨论（30-60 分钟）
         └── 投票表决
         ↓
Step 4: 决策结果
         ├── 通过（Approve）
         ├── 有条件通过（Conditional Approve）
         ├── 否决（Reject）
         └── 暂缓（Defer / Go Back）
```

### IC Memo 结构（投资备忘录）

IC Memo 是投决会的核心文件，结构化地呈现项目评估的全部关键信息。

#### 标准 IC Memo 目录

| 章节 | 内容 | 篇幅（页） |
|------|------|-----------|
| 1. 执行摘要 | 投资亮点、关键风险、估值和条款、投资建议 | 1-2 |
| 2. 公司概况 | 历史沿革、业务模式、产品/服务、组织架构 | 2-3 |
| 3. 市场分析 | 市场规模、增长趋势、竞争格局、行业政策 | 3-5 |
| 4. 竞争分析 | 竞争对手对比、竞争优势、护城河 | 2-3 |
| 5. 财务分析 | 历史财务、关键指标、财务预测、敏感性分析 | 3-5 |
| 6. 团队评估 | 创始人/管理层背景、团队完整性、激励机制 | 1-2 |
| 7. 估值分析 | 估值方法、可比公司、可比交易、估值区间 | 2-3 |
| 8. 投资条款 | Term Sheet 核心条款、条款对比分析 | 1-2 |
| 9. 风险分析 | 主要风险及缓释措施 | 1-2 |
| 10. 退出规划 | 退出路径、退出时间、退出回报预估 | 1 |
| 11. 投后管理计划 | 投后增值服务、监控指标、里程碑 | 1 |
| **合计** | | **20-30** |

### 打分卡 / 评估矩阵（Scorecard）

打分卡是将主观评估标准化的工具，确保不同项目之间的可比性。

#### VC 项目评估打分卡示例

| 评估维度 | 权重 | 评分标准（1-5分） | 本项目得分 |
|----------|------|-------------------|-----------|
| **市场空间** | 20% | 5: TAM>1000亿 4: 500-1000亿 3: 100-500亿 2: 50-100亿 1: <50亿 | |
| **团队质量** | 25% | 5: 连续成功创业者 4: 行业资深+互补团队 3: 有经验团队 2: 单一创始人 1: 无相关经验 | |
| **产品/技术** | 20% | 5: 颠覆性技术 4: 明显差异化 3: 有特色 2: 同质化 1: 落后 | |
| **商业模式** | 15% | 5: 已验证且高效 4: 基本验证 3: 有初步验证 2: 待验证 1: 不清晰 | |
| **竞争壁垒** | 10% | 5: 强网络效应/品牌 4: 技术壁垒 3: 规模效应 2: 弱壁垒 1: 无壁垒 | |
| **估值合理性** | 10% | 5: 明显低估 4: 合理 3: 偏高但可接受 2: 过高 1: 严重过高 | |
| **加权总分** | 100% | | |

**评分阈值：**
- 4.0+：强烈推荐投资
- 3.5-4.0：推荐投资
- 3.0-3.5：谨慎推荐，需关注特定风险
- <3.0：不建议投资

### 决策层级

大型基金通常采用分级决策机制：

| 投资金额 | 决策层级 | 审批流程 |
|----------|----------|----------|
| <500 万 | 投资总监 + 合伙人 | 简化审批 |
| 500 万-2,000 万 | IC 简会 | 快速 IC（周会附议） |
| 2,000 万-5,000 万 | 标准 IC | 完整 IC Memo + IC 会议 |
| >5,000 万 | IC + LPAC 咨询 | 完整流程 + LPAC 通报 |

### 投决会的常见挑战

| 挑战 | 表现 | 应对措施 |
|------|------|----------|
| 群体思维 | IC 成员趋向一致，忽视独立判断 | 设置"魔鬼代言人"角色 |
| 锚定效应 | 过度依赖初始估值或先入为主的判断 | 多维度独立估值 |
| 沉没成本 | 因已投入的 DD 时间和费用而勉强通过 | 明确"不投资也是好决策" |
| FOMO | 担心错过热门项目而仓促决策 | 回归基金策略和评估标准 |
| 关系投资 | 因人情关系而非项目质量投资 | 坚持客观评估标准 |

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 投资决策委员会 | Investment Committee (IC) | 负责投资决策的最高权力机构 |
| 投资备忘录 | IC Memo / Investment Memo | 投决会审阅的项目评估核心文件 |
| 打分卡 | Scorecard / Evaluation Matrix | 将项目评估标准化的量化工具 |
| 有条件通过 | Conditional Approval | 附带前提条件的审批通过 |
| 魔鬼代言人 | Devil's Advocate | 在 IC 中专门负责质疑和挑战的角色 |
| 投资策略 | Investment Strategy / Mandate | 基金的投资方向、阶段、行业等限制条件 |
| 投后管理计划 | Post-Investment Plan | 投资完成后的增值服务和监控安排 |

## 实务要点

- **IC Memo 质量决定决策质量：** 好的 IC Memo 应该信息充分、逻辑清晰、结论明确，项目团队应在 IC 会前花足够时间打磨 Memo
- **投决会不是"走过场"：** 如果 IC 会只是项目团队的"汇报会"而没有真正的挑战和讨论，说明决策机制存在问题
- **建立"不投资"的文化：** 优秀的基金应该让团队成员敢于在 IC 上说"不"，避免"只投不拒"的惯性
- **定期回顾投资决策：** 每半年回顾一次过去的投决记录，分析哪些决策是正确的、哪些是错误的，持续优化决策标准
- **关注"非共识"机会：** 最好的投资往往是 IC 中最具争议的项目，完全一致通过的项目可能已被市场充分定价

## 常见误区

- **将 IC Memo 当作"推销文档"：** IC Memo 的目的是帮助 IC 做出最优决策，而不是说服 IC 通过项目，过度美化项目会导致决策偏差
- **忽视"不投资"的机会成本：** 每投资一个项目意味着放弃了其他投资机会，决策时应考虑"这笔钱是否应该投到其他地方"
- **打分卡变成形式主义：** 打分卡的价值在于结构化思考，而非追求精确的分数，如果打分流于形式则失去了意义
