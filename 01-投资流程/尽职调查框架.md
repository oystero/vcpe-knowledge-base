# 尽职调查框架

## 概述
尽职调查（Due Diligence, DD）是 VC/PE 投资中对目标企业进行全面、系统评估的核心环节。DD 的目标是验证投资假设、发现潜在风险、为估值和条款谈判提供依据。高质量的 DD 能显著降低投资失败率，是投资决策最重要的信息输入。

## 核心内容

### DD 的四大维度

| 维度 | 英文 | 核心目标 | 典型执行方 |
|------|------|----------|-----------|
| 财务尽调 | Financial DD | 验证财务数据真实性、评估盈利能力 | 会计师事务所（四大/内资所） |
| 法律尽调 | Legal DD | 发现法律风险、验证合规性 | 律师事务所 |
| 商业尽调 | Commercial DD | 验证市场空间、竞争地位、商业模式 | 投资团队 + 第三方咨询 |
| 技术尽调 | Technology DD | 评估技术壁垒、产品成熟度、技术团队 | 技术专家 / CTO 级顾问 |

### 财务尽调（Financial DD）

**核心关注点：**

| 评估项 | 具体内容 | 常见发现 |
|--------|----------|----------|
| 收入真实性 | 收入确认政策、大客户集中度、关联交易 | 虚假收入、提前确认收入 |
| 盈利质量 | 调整后 EBITDA、Non-recurring items | 一次性收益虚增利润 |
| 现金流 | 经营性现金流与利润的匹配度 | 利润高但现金流差（应收高企） |
| 资产质量 | 存货周转、应收账款账龄、固定资产状况 | 坏账、存货减值风险 |
| 负债结构 | 有息负债、或有负债、表外负债 | 隐性担保、关联方借款 |
| 税务合规 | 税收优惠政策合法性、税务风险 | 偷漏税、政策到期风险 |

**财务 DD 的典型流程：**
1. 签署 NDA，获取财务数据（3 年审计报告 + 最近一期管理报表）
2. 会计师事务所出具财务 DD 报告（通常 4-6 周）
3. 投资团队审阅报告，与管理层讨论发现
4. 调整估值模型（如有重大发现）

### 法律尽调（Legal DD）

**核心关注点：**

| 评估项 | 具体内容 | 常见风险 |
|--------|----------|----------|
| 公司设立与存续 | 工商登记、公司章程、历次变更 | 设立程序瑕疵 |
| 股权结构 | 股东名册、股权代持、股权质押 | 代持纠纷、股权不清晰 |
| 重大合同 | 客户合同、供应商合同、租赁合同 | 不利条款、违约风险 |
| 知识产权 | 专利、商标、著作权、域名 | IP 归属争议、专利无效风险 |
| 劳动人事 | 劳动合同、竞业限制、社保公积金 | 劳动纠纷、核心人员流失风险 |
| 诉讼仲裁 | 在诉案件、历史诉讼、行政处罚 | 重大诉讼风险 |
| 合规经营 | 行业许可证、资质证书、数据合规 | 无证经营、数据泄露风险 |
| 关联交易 | 关联方关系、关联交易定价公允性 | 利益输送、损害公司利益 |

**法律 DD 报告结构：**
- 摘要（Executive Summary）：重大风险事项和建议
- 详细发现（Detailed Findings）：按法律领域分类
- 风险等级评定：高/中/低风险
- 整改建议和交割前提条件

### 商业尽调（Commercial DD）

**核心评估框架：**

#### 1. 市场分析

| 分析维度 | 关键问题 | 数据来源 |
|----------|----------|----------|
| 市场规模 | TAM/SAM/SOM 分别是多少？ | 行业报告、专家访谈 |
| 增长驱动 | 市场增长的核心驱动力是什么？ | 行业趋势、政策分析 |
| 市场阶段 | 市场处于萌芽/成长/成熟/衰退期？ | 增长率、渗透率 |
| 进入壁垒 | 新进入者的门槛有多高？ | 技术、品牌、规模、政策 |

#### 2. 竞争分析

| 分析维度 | 关键问题 | 分析工具 |
|----------|----------|----------|
| 竞争格局 | 市场集中度如何？CR5 是多少？ | 波特五力、竞争矩阵 |
| 竞争优势 | 目标企业的核心竞争力是什么？ | 护城河分析 |
| 替代风险 | 是否有替代产品/技术的风险？ | 技术路线分析 |
| 定价能力 | 企业是否有定价权？ | 价格弹性分析 |

#### 3. 客户/用户分析

| 分析维度 | 关键问题 |
|----------|----------|
| 客户集中度 | 前 5 大客户收入占比是否过高（>50% 为警示）？ |
| 客户留存 | NDR（Net Dollar Retention）是否 >100%？ |
| 获客成本 | CAC 回收期（Payback Period）是否合理？ |
| NPS / 满意度 | 客户满意度和推荐意愿如何？ |

#### 4. 商业尽调的专家访谈（Expert Call）

- 行业专家访谈：通常 5-10 位，通过 GLG、Bain Expert、Capvision 等专家网络安排
- 客户访谈：3-5 位核心客户，了解产品满意度和续约意愿
- 竞争对手分析：2-3 位竞争对手的前员工或行业观察者
- 供应商访谈：2-3 位关键供应商，了解合作稳定性和议价关系

### 技术尽调（Technology DD）

**核心评估项：**

| 评估项 | 具体内容 | 评估方法 |
|--------|----------|----------|
| 技术架构 | 系统架构的可扩展性、稳定性 | 技术文档审阅 + 架构评审 |
| 代码质量 | 代码规范、测试覆盖率、技术债务 | 代码审计（静态分析 + 人工审查） |
| 技术壁垒 | 核心技术的先进性和可复制性 | 专利分析 + 专家评估 |
| 产品成熟度 | 产品功能完整性、用户体验 | 产品 Demo + 用户测试 |
| 安全合规 | 数据安全、隐私保护、合规认证 | 安全审计 + 合规检查 |
| 技术团队 | 核心技术人员的背景和稳定性 | 团队访谈 + 历史贡献评估 |

**技术 DD 的特殊注意事项：**
- 对于 AI/ML 项目：重点关注训练数据来源、模型性能、可解释性
- 对于硬件项目：关注供应链稳定性、BOM 成本、量产可行性
- 对于 SaaS 项目：关注架构可扩展性、多租户设计、SLA 达成率

### DD 流程与时间线

```
Week 1-2:  签署 NDA → 收集资料 → 组建 DD 团队
Week 2-3:  财务/法律 DD 启动 → 商业 DD 的专家访谈安排
Week 3-4:  各维度 DD 并行推进 → 初步发现汇总
Week 4-5:  DD 报告初稿 → 管理层回应讨论
Week 5-6:  最终 DD 报告 → 投决会准备
```

**典型 DD 时间线（按项目类型）：**

| 项目类型 | DD 时长 | 说明 |
|----------|---------|------|
| 早期 VC（种子/A轮） | 2-4 周 | 重点在团队和产品，财务 DD 较轻 |
| 成长期 VC（B-D轮） | 4-6 周 | 全面 DD，但深度可适当控制 |
| 成长型 PE | 6-8 周 | 深度 DD，需详细的财务模型 |
| Buyout PE | 8-12 周 | 最全面的 DD，包括运营 DD |

### DD 成本

| DD 类型 | 典型费用（人民币） | 说明 |
|----------|------------------|------|
| 财务 DD | 30-100 万 | 四大事务所 vs 内资所差异大 |
| 法律 DD | 20-80 万 | 取决于项目复杂度和律师所级别 |
| 商业 DD | 15-50 万 | 专家访谈费用 + 第三方咨询 |
| 技术 DD | 10-30 万 | 通常由内部团队 + 外部专家完成 |
| **合计** | **75-260 万** | 通常由被投企业承担或投资方承担 |

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 尽职调查 | Due Diligence (DD) | 投资前对目标企业进行全面调查和评估的过程 |
| 保密协议 | NDA (Non-Disclosure Agreement) | 保护 DD 过程中获取的商业秘密的法律协议 |
| 调整后 EBITDA | Adjusted EBITDA | 剔除一次性项目后的息税折旧摊销前利润 |
| 净收入留存率 | NDR (Net Dollar Retention) | 存量客户的收入留存和扩展情况 |
| 获客成本 | CAC (Customer Acquisition Cost) | 获取一个新客户的平均成本 |
| 总可触达市场 | TAM (Total Addressable Market) | 产品/服务的全部潜在市场空间 |
| 保底条款 | Material Adverse Change (MAC) | DD 发现重大不利变化时的保护条款 |

## 实务要点

- **DD 不是走过场：** DD 的目的是发现"交易杀手"（Deal Killer），应保持独立判断，不要因为沉没成本（已投入的 DD 时间和费用）而忽视重大风险
- **关注"软信息"：** 除了硬数据，管理层的诚信度、配合度、对问题的回应方式等"软信息"往往比数字更能预示投资结果
- **专家访谈要提前准备：** 商业 DD 中的专家访谈质量高度依赖于问题设计，应提前研究行业背景、准备结构化问题清单
- **DD 发现要在估值中体现：** 重大风险发现应通过估值调整（Valuation Adjustment）或保护性条款（如对赌、Escrow）来体现，而非简单"接受风险"
- **早期项目可以简化 DD：** 对于种子/A轮项目，重点应放在团队背景调查、产品技术验证和客户访谈上，深度财务 DD 的意义有限

## 常见误区

- **过度依赖 DD 报告而忽视独立判断：** DD 报告是信息输入而非决策本身，投资团队应保持独立思考，特别是当 DD 发现与管理层表述矛盾时
- **只做"验证性" DD 而非"发现性" DD：** 很多 DD 只是在验证已有的投资假设，而忽视了主动发现风险，好的 DD 应该能回答"什么情况下这笔投资会失败"
- **忽视交叉验证：** 单一信息源的可靠性有限，关键发现应通过多个独立来源交叉验证（如客户访谈 + 财务数据 + 行业报告）
