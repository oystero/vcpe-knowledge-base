# 商业尽调方法

## 概述
商业尽职调查（Commercial Due Diligence, CDD）是对目标公司所处市场、竞争环境、客户基础和商业模式进行独立验证的过程。商业尽调的核心目的是回答一个根本性问题：这家公司的业务是否真的如其呈现的那样好？它通过第三方视角验证管理层声称的市场机会和增长潜力，为投资决策提供独立的商业判断依据。

## 核心内容

### 一、市场规模验证（TAM / SAM / SOM）

#### 概念框架

| 层级 | 英文 | 定义 | 估算方法 |
|------|------|------|---------|
| 潜在市场总量 | TAM (Total Addressable Market) | 假设100%市场渗透率下的最大市场规模 | 自上而下（Top-down）+ 自下而上（Bottom-up）交叉验证 |
| 可服务市场 | SAM (Serviceable Addressable Market) | 公司产品/服务能够触达的市场部分 | TAM × 地域覆盖率 × 产品匹配度 |
| 可获得市场 | SOM (Serviceable Obtainable Market) | 公司在短期内可实际获取的市场份额 | SAM × 预期市场份额 |

#### 验证 Checklist

- [ ] 使用自上而下方法：从行业报告获取宏观市场规模
- [ ] 使用自下而上方法：从客户数量、单价、频次推算
- [ ] 交叉验证两种方法的结论，差异不超过30%为合理
- [ ] 检查行业报告的数据来源和预测假设
- [ ] 对标相似市场的发展轨迹（如中国 vs 美国市场）
- [ ] 验证市场的增长驱动力是否可持续
- [ ] 评估政策环境对市场规模的影响

#### TAM估算的常见错误

| 错误类型 | 示例 | 纠正方法 |
|---------|------|---------|
| 定义过宽 | "中国所有企业都是我们的潜在客户" | 明确目标客户画像和使用场景 |
| 忽视替代方案 | "我们是唯一解决方案" | 评估客户的替代选择和转换成本 |
| 低估市场成熟度 | "这是一个全新的蓝海市场" | 寻找类似市场的发展阶段参考 |
| 忽视价格约束 | 假设所有潜在客户都愿意支付当前价格 | 评估价格弹性对市场规模的影响 |

### 二、竞争格局分析

#### 分析框架

##### Porter五力模型应用

| 力量 | 评估维度 | 关键问题 |
|------|---------|---------|
| 现有竞争者 | 市场集中度、竞争强度 | 行业CR5是多少？价格战是否频繁？ |
| 潜在进入者 | 进入壁垒高低 | 新玩家进入需要多少资金和时间？ |
| 替代品威胁 | 替代方案的性价比 | 客户是否可以用其他方式解决同一需求？ |
| 供应商议价力 | 供应商集中度 | 关键原材料/技术是否被少数供应商控制？ |
| 客户议价力 | 客户集中度、转换成本 | 客户是否可以轻易切换到竞争对手？ |

##### 竞争对手分析 Checklist

- [ ] 列出直接竞争对手（产品/服务相同）和间接竞争对手（替代方案）
- [ ] 分析竞争对手的融资历史、估值、团队背景
- [ ] 对比产品功能、定价策略、客户群体
- [ ] 评估竞争对手的市场份变化趋势
- [ ] 分析竞争壁垒的可持续性（技术壁垒、网络效应、品牌壁垒、规模效应）
- [ ] 识别潜在的跨界竞争者（如大厂进入）

#### 竞争优势评估矩阵

| 壁垒类型 | 持久性 | 建立难度 | 典型行业 |
|---------|--------|---------|---------|
| 网络效应 | 极高 | 极高 | 社交平台、支付网络 |
| 规模经济 | 高 | 高 | 制造业、物流 |
| 品牌壁垒 | 高 | 高 | 消费品、奢侈品 |
| 技术壁垒 | 中 | 中 | 半导体、生物科技 |
| 数据壁垒 | 中-高 | 中 | AI、数据服务 |
| 转换成本 | 中 | 低 | SaaS、ERP系统 |
| 监管壁垒 | 高 | 高 | 金融、医疗 |

### 三、客户访谈方法

#### 访谈设计

##### 访谈对象选择

| 对象类型 | 数量建议 | 访谈重点 |
|---------|---------|---------|
| 现有核心客户 | 5-8家 | 满意度、续约意愿、价格敏感度 |
| 现有普通客户 | 5-10家 | 使用场景、替代方案、推荐意愿 |
| 流失客户 | 3-5家 | 流失原因、竞争对手吸引力 |
| 潜在客户 | 5-8家 | 需求痛点、购买决策因素、预算分配 |
| 行业专家 | 3-5位 | 行业趋势、竞争格局、技术方向 |

##### 访谈问题设计原则

- **开放式问题优先**：避免引导性问题，让受访者自由表达
- **具体化**：用"请描述您上一次采购决策的过程"替代"您对我们的产品满意吗？"
- **量化验证**：询问具体的数字（预算、使用频次、价格区间）
- **对比视角**：让受访者对比不同供应商的优劣

#### 核心访谈问题模板

##### 针对现有客户

```
1. 您最初是如何了解到 [公司名称] 的？选择它的主要原因是？
2. 在使用过程中，最满意和最不满意的地方分别是什么？
3. 如果明天 [公司名称] 不存在了，您会用什么替代方案？
4. 您是否向同行推荐过我们的产品？为什么？
5. 未来12个月，您对 [产品类别] 的预算预计会如何变化？
6. 如果竞争对手提供类似产品但便宜20%，您会考虑更换吗？
```

##### 针对流失客户

```
1. 您停止使用 [公司名称] 的主要原因是什么？
2. 您现在使用什么替代方案？与之前相比体验如何？
3. 如果 [公司名称] 解决了您当时的问题，您会考虑重新使用吗？
4. 您认为 [公司名称] 最需要改进的三个方面是什么？
```

##### 针对行业专家（Expert Call）

```
1. 您如何看待 [行业名称] 未来3-5年的发展趋势？
2. 您认为这个赛道的终局格局会是什么样的？
3. 在这个领域，什么样的公司最有可能胜出？关键成功因素是什么？
4. 您如何评价 [目标公司] 在行业中的地位和竞争力？
5. 这个行业最大的风险和不确定性是什么？
```

### 四、渠道验证

#### 渠道类型分析

| 渠道类型 | 验证重点 | 关键指标 |
|---------|---------|---------|
| 直销渠道 | 销售团队效率、获客成本 | CAC、销售周期、转化率 |
| 分销渠道 | 渠道覆盖、渠道忠诚度 | 渠道覆盖率、渠道流失率 |
| 线上渠道 | 流量来源、转化漏斗 | DAU/MAU、转化率、留存率 |
| 合作伙伴 | 合作深度、合作稳定性 | 合作收入占比、续约率 |

#### 渠道验证 Checklist

- [ ] 核实前十大客户的实际采购量与公司声称是否一致
- [ ] 验证分销商/代理商的库存水平和动销情况
- [ ] 检查渠道是否存在压货（Channel Stuffing）迹象
- [ ] 评估渠道的多样性和集中度风险
- [ ] 核查线上渠道的流量数据（SimilarWeb、App Annie等）
- [ ] 验证获客成本（CAC）的计算口径和趋势变化
- [ ] 评估客户生命周期价值（LTV）与CAC的比率

#### LTV/CAC分析

| 指标 | 计算公式 | 健康标准 |
|------|---------|---------|
| LTV | ARPU × 客户生命周期 × 毛利率 | 行业差异大 |
| CAC | 销售和营销总成本 / 新增客户数 | 趋势应下降 |
| LTV/CAC | LTV ÷ CAC | > 3.0（理想状态） |
| CAC回收期 | CAC ÷ (月ARPU × 毛利率) | < 12个月 |

### 五、行业专家访谈（Expert Call）

#### 专家来源渠道

| 来源 | 优势 | 劣势 |
|------|------|------|
| 专家网络（GLG、凯盛、Guidepoint） | 快速匹配、质量可控 | 费用高、合规风险 |
| 行业会议/论坛 | 免费、直接接触 | 效率低、随机性大 |
| 学术机构 | 深度分析、理论框架 | 可能脱离实际 |
| 前从业者 | 实操经验丰富 | 可能有偏见 |
| 上下游合作伙伴 | 视角独特 | 利益相关性 |

#### Expert Call最佳实践

1. **准备充分**：提前准备问题清单，按优先级排序
2. **时间控制**：45-60分钟为宜，前15分钟用于建立信任和背景了解
3. **录音合规**：确认是否可以录音，遵守各地法律法规
4. **交叉验证**：同一问题至少询问3位独立专家，取共识观点
5. **关注分歧点**：专家意见分歧的地方往往是投资决策的关键
6. **合规注意**：避免询问可能涉及内幕信息的问题

#### 专家访谈记录模板

```
访谈日期：YYYY-MM-DD
专家背景：[行业/职位/年限]
访谈时长：XX分钟
访谈人：[姓名]

核心观点：
1. [观点1]
2. [观点2]
3. [观点3]

与管理层声称的一致之处：
- [要点]

与管理层声称的分歧之处：
- [要点]

对投资决策的启示：
- [结论]
```

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| TAM | Total Addressable Market | 假设100%市场渗透率下的最大市场规模 |
| SAM | Serviceable Addressable Market | 公司产品/服务能够触达的市场部分 |
| SOM | Serviceable Obtainable Market | 公司在短期内可实际获取的市场份额 |
| CAC | Customer Acquisition Cost | 获取一个新客户的平均成本 |
| LTV | Lifetime Value | 客户生命周期内为公司创造的总价值 |
| ARPU | Average Revenue Per User | 每用户平均收入 |
| Expert Call | Expert Network Call | 通过专家网络平台安排的行业专家访谈 |
| NPS | Net Promoter Score | 净推荐值，衡量客户忠诚度和满意度的指标 |
| Cohort Analysis | 群组分析 | 按用户获取时间分组分析其行为和价值变化 |
| Market Sizing | 市场规模测算 | 估算特定市场总容量的过程 |

## 实务要点

1. **独立验证优先**：商业尽调的核心价值在于独立于管理层视角的验证。不要仅依赖管理层提供的PPT和数据，要通过客户访谈、专家访谈、公开数据等多种渠道独立核实。

2. **TAM要自下而上验证**：管理层呈现的TAM通常是自上而下（Top-down）的宏观估算。投资人应要求管理层提供自下而上（Bottom-up）的推算逻辑，并独立验证关键假设。

3. **关注客户的"真实行为"而非"声称态度"**：客户访谈中，行为数据比态度表达更可靠。询问"您上次采购的决策过程"比"您觉得我们的产品好吗"更有价值。NPS评分需要结合实际续约率和增购行为。

4. **竞争分析要动态视角**：市场竞争格局不是静态的。要评估当前竞争者、潜在进入者、替代方案三重威胁。特别关注大厂（BAT、TMD等）是否会进入赛道。

5. **渠道验证是商业尽调的核心环节**：通过直接接触客户的终端用户和渠道合作伙伴，可以发现管理层报告中未提及的问题。渠道库存水平、终端动销数据是最真实的业务健康指标。

## 常见误区

1. **过度依赖第三方行业报告**：行业报告（如艾瑞、IDC、Gartner）提供的是宏观估算，往往存在数据来源不透明、预测过于乐观、定义口径不一致等问题。投资人应将行业报告作为参考起点，而非最终结论，需结合自下而上的独立测算。

2. **忽视"市场规模陷阱"**：大市场（TAM大）不等于好机会。在充分竞争的大市场中，小公司可能难以获得有意义的市场份额。相反，小但快速增长的细分市场可能提供更好的投资机会。关键是SOM而非TAM。

3. **专家访谈的选择偏差**：专家网络平台提供的专家可能与目标公司存在利益关系（如前员工、合作伙伴），或者其观点可能不代表行业主流。需要评估专家的独立性，并交叉验证多个来源的观点。
